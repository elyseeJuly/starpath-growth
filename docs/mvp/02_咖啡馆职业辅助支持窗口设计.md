# 咖啡馆职业辅助支持窗口设计 v1.0

## 一、定位与目标

**定位**：面向公益咖啡馆场景的轻量级辅助工具，帮助康复老师记录患者工作表现，支持能力评估和干预建议。

**目标**：
- 实时记录患者工作数据
- 辅助康复老师提供个性化指导
- 为能力画像提供真实数据源

---

## 二、用户角色与使用场景

### 2.1 用户角色
| 角色 | 职责 | 需求 |
|------|------|------|
| **康复老师** | 指导患者工作、记录数据 | 便捷的数据录入、实时反馈 |
| **患者** | 完成工作任务 | 清晰的任务指引、及时的提示 |
| **管理员** | 查看汇总数据、管理用户 | 数据统计、用户管理 |

### 2.2 使用场景
```
工作中：
  ↓
[康复老师] 记录任务执行情况
[患者] 接收任务指引
  ↓
工作结束：
  ↓
[康复老师] 填写当日总结
[系统] 更新患者能力画像
```

## 三、功能模块设计

### 3.1 任务管理模块

#### 3.1.1 任务列表
| 任务名称 | 任务描述 | 步骤数 |
|---------|---------|-------|
| 接待顾客 | 听取订单、确认需求 | 3步 |
| 制作咖啡 | 操作咖啡机、完成制作 | 5步 |
| 清洁场地 | 打扫卫生、整理物品 | 4步 |
| 清洗杯具 | 清洗、消毒、摆放 | 3步 |

#### 3.1.2 任务步骤示例（制作咖啡）
```
1. 选择咖啡类型 → 2. 按下对应按钮 → 3. 等待萃取 → 4. 倒入杯子 → 5. 交给顾客
```

### 3.2 数据记录模块

#### 3.2.1 实时记录表单
| 字段 | 类型 | 说明 |
|------|------|------|
| 任务类型 | 下拉选择 | 选择当前任务 |
| 完成步骤 | 数字输入 | 已完成的步骤数 |
| 干预次数 | 数字输入 | 老师干预次数 |
| 完成质量 | 1-5评分 | 任务完成质量 |
| 情绪状态 | 选择 | 平静/焦虑/愉快/烦躁 |
| 备注 | 文本 | 其他说明 |

#### 3.2.2 顾客交互记录
| 字段 | 类型 | 说明 |
|------|------|------|
| 指令理解度 | 1-5评分 | 理解顾客需求的程度 |
| 眼神交流 | 0/1 | 是否有适当眼神接触 |
| 回应准确性 | 0/1 | 回应是否正确 |
| 社交主动性 | 0/1 | 是否主动交流 |

### 3.3 实时提示模块

#### 3.3.1 任务提示
- 当前任务步骤指引
- 下一步操作提示
- 常见错误提醒

#### 3.3.2 社交提示
- 与顾客交流的话术建议
- 眼神交流提醒
- 情绪调节建议

### 3.4 数据展示模块

#### 3.4.1 实时状态面板
```
患者：张三
日期：2024-01-15
今日任务完成：3/5
累计干预次数：5
当前情绪：平静
```

#### 3.4.2 能力趋势图
- 按日期展示各能力维度变化
- 任务完成率趋势
- 干预次数趋势

### 3.5 干预建议模块

#### 3.5.1 实时建议
基于当前数据自动生成：
- 下一步干预策略
- 重点训练方向
- 休息建议

#### 3.5.2 历史干预记录
查看过往干预策略和效果

## 四、界面设计

### 4.1 主界面布局

```
┌──────────────────────────────────────────────────────┐
│  顶部：患者信息 + 日期选择                            │
├───────────────┬─────────────────────────────────────┤
│ 左侧：任务列表 │ 右侧：记录表单 + 实时提示           │
│              │                                     │
│  • 接待顾客   │  [任务记录表单]                     │
│  • 制作咖啡   │                                     │
│  • 清洁场地   │  [实时提示区域]                     │
│  • 清洗杯具   │                                     │
├───────────────┴─────────────────────────────────────┤
│ 底部：能力趋势图表                                   │
└──────────────────────────────────────────────────────┘
```

### 4.2 界面设计要点
- **简洁直观**：避免复杂操作，适合康复老师快速录入
- **大字体**：便于查看
- **颜色区分**：不同任务状态用不同颜色
- **一键操作**：常用操作尽量简化

## 六、技术实现

### 6.1 前端技术栈
- **框架**：React + TypeScript
- **UI库**：Tailwind CSS 3
- **图表**：ECharts
- **状态管理**：React Context

### 6.2 后端接口
| 接口 | 方法 | 用途 |
|------|------|------|
| `/api/tasks` | GET | 获取任务列表 |
| `/api/records` | POST | 提交记录 |
| `/api/records/{patientId}` | GET | 获取患者记录 |
| `/api/profile/{patientId}` | GET | 获取患者画像 |
| `/api/suggestion` | POST | 获取干预建议 |

### 6.3 数据模型（扩展）

```javascript
// 任务记录扩展
TaskRecord {
  id: UUID
  patientId: UUID
  taskType: String
  stepsCompleted: Number
  totalSteps: Number
  interventionCount: Number
  qualityScore: 1-5
  emotionState: String
  createdAt: DateTime
}
```

## 八、与生态平台的关系

```
咖啡馆窗口 (真实数据采集)
       ↓
       | API对接
       ↓
多模型生态平台 (画像生成、干预建议)
       ↓
       | 数据回流
       ↓
咖啡馆窗口 (个性化指导)
```

**数据流向**：
1. 咖啡馆窗口采集真实工作数据
2. 数据同步到生态平台
3. 平台生成能力画像和干预建议
4. 建议回流到咖啡馆窗口指导实际工作
