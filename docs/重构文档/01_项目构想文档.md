## 一、项目定义

### 1.1 一句话定义

**StarPath** 是一套面向成年ASD（自闭症谱系障碍）患者的**社会化辅助支持系统**，通过"AI能力画像 + 多智能体协作 + 多方数据融合"构建患者成长支持网络。

### 1.2 终极愿景

让 StarPath 成为像**盲道**、**AED**一样的社会基础设施——任何社区只要有ASD患者，就能接入StarPath获得支持。

### 1.3 目标用户

| 角色 | 核心诉求 | 使用频率 |
|------|---------|---------|
| **ASD患者** | 提升社交技能、适应工作岗位、改善生活质量 | 每日 |
| **家长/家属** | 了解孩子状态、辅助训练、跟踪成长 | 每日 |
| **康复老师/社工** | 评估能力、管理训练、制定方案 | 工作日 |
| **用人单位/社区** | 岗位适配、环境调整、工作支持 | 不定期 |
| **系统管理员** | 用户管理、系统运维、数据分析 | 按需 |

## 二、问题域分析

### 2.1 核心痛点

```
痛点1：ASD患者沟通困难 → 传统评估方式主观、片面
痛点2：家属/老师无法全面了解患者 → 干预碎片化、缺乏连续性
痛点3：多方信息孤岛 → 家庭、机构、社区之间数据不互通
痛点4：缺乏个性化 → 千人一面的训练方案无法适应个体差异
```

### 2.2 问题本质

ASD患者的支持不是"单一技能训练"的问题，而是一个**跨角色、跨场景、跨时间的系统工程**。现有方案试图用人来解决系统性问题，必然失败。需要一套以AI为核心的数据驱动支持系统。

### 2.3 解决的问题

| 问题 | StarPath的解法 |
|------|---------------|
| 评估主观 | 多源数据融合，客观量化能力画像 |
| 信息孤岛 | 统一数据平台，四方角色数据互通 |
| 干预碎片化 | 基于画像的个性化、持续性干预方案 |
| 缺乏个性化 | AI驱动的自适应学习路径 |

## 四、多智能体框架设计

### 4.1 智能体定义

```
                    ┌─────────────────┐
                    │   协调智能体     │
                    │   Orchestrator  │
                    │  (任务分发+调度) │
                    └────────┬────────┘
                             │
        ┌────────────────────┼────────────────────┐
        ↓                    ↓                    ↓
┌───────────────┐    ┌───────────────┐    ┌───────────────┐
│ 评估智能体    │    │ 干预智能体    │    │ 反馈智能体    │
│ Evaluator     │    │ Intervenor    │    │ Reflector     │
│               │    │               │    │               │
│ • 多源融合    │    │ • 方案生成    │    │ • 效果评估    │
│ • 画像更新    │    │ • 实时提示    │    │ • 策略优化    │
│ • 风险预警    │    │ • 路径规划    │    │ • 模型迭代    │
└───────┬───────┘    └───────┬───────┘    └───────┬───────┘
        │                    │                    │
        ↓                    ↓                    ↓
┌───────────────────────────────────────────────────────────┐
│                      数据层                               │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐        │
│  │患者数据 │ │家庭数据 │ │教育数据 │ │社会数据 │        │
│  └─────────┘ └─────────┘ └─────────┘ └─────────┘        │
└───────────────────────────────────────────────────────────┘
```

### 4.2 智能体职责

#### 协调智能体（Orchestrator）
- 接收各端输入事件
- 决定触发哪个子智能体
- 管理智能体间的数据流转
- 维护任务优先级队列

#### 评估智能体（Evaluator）
- **输入**：行为记录、任务数据、生理指标、主观报告
- **处理**：多源数据融合算法、加权评分
- **输出**：能力画像更新、风险预警、变化趋势

#### 干预智能体（Intervenor）
- **输入**：当前能力画像、任务上下文、用户角色
- **处理**：基于画像的个性化方案生成
- **输出**：训练计划、实时提示、步骤引导

#### 反馈智能体（Reflector）
- **输入**：干预效果数据、用户反馈、画像变化
- **处理**：效果归因分析、策略A/B对比
- **输出**：干预策略优化建议、模型参数调整

### 4.3 智能体协作流程

```
事件触发（患者完成任务）
    │
    ↓
协调智能体  ← 接收事件
    │
    ├→ 评估智能体  ← 评估当前表现，更新画像
    │       │
    │       └→ 返回：画像快照、变化量
    │
    ├→ 干预智能体  ← 基于新画像生成后续建议
    │       │
    │       └→ 返回：下一步训练计划
    │
    └→ 反馈智能体  ← 对比历史，评估策略有效性
            │
            └→ 返回：策略调整建议（异步）
```

## 五、数据采集体系

### 5.1 四源数据模型

```
数据来源                 数据类型              采集方式
─────────────────────────────────────────────────────
患者本人  ──────────→  任务完成记录          ← 手动输入
                       情绪状态打卡            ← 手动选择
                       自我能力评估            ← 问卷表单

家庭方   ──────────→  日常行为观察            ← 手动记录
                       家庭训练数据            ← 手动输入
                       异常事件报告            ← 手动记录

教育方   ──────────→  康复训练数据            ← 专业录入
（康复机构/社工）     标准化评估结果          ← 专业录入
                       干预记录                ← 专业录入
                       教学目标达成            ← 系统自动

社会方   ──────────→  岗位表现数据            ← 系统自动
（工作单位/社区）     出勤记录                ← 系统自动
                       工作质量评估            ← 主管录入
                       环境适应度              ← 观察录入

外部设备  ─────────→  心率/HRV                ← 可穿戴设备
（未来接入）         皮肤电反应              ← 传感器
                       运动数据                ← 加速度计
                       语音交互记录            ← 麦克风
```

### 5.2 数据采集字段规范

| 字段名 | 类型 | 必填 | 采集方式 | 来源角色 | 频率 |
|--------|------|------|---------|---------|------|
| task_completion | int | ✅ | 手动/自动 | 患者/教育方 | 每次任务 |
| emotion_state | enum | ✅ | 手动选择 | 患者 | 每日至少1次 |
| intervention_count | int | ✅ | 手动录入 | 教育方 | 每次干预 |
| quality_score | int(1-5) | ✅ | 手动录入 | 教育方/社会方 | 每次任务 |
| heart_rate | int | ❌ | 设备采集 | 可穿戴设备 | 实时 |
| gsr | float | ❌ | 设备采集 | 传感器 | 实时 |
| note | text | ❌ | 手动输入 | 全部角色 | 按需 |

## 六、能力画像引擎

### 6.1 画像模型（保留v1.0设计并增强）

8大核心维度（与审计报告一致），增加：

| 增强项 | 说明 |
|--------|------|
| **画像版本控制** | 每次更新生成新版本快照，支持历史回溯 |
| **置信度评分** | 基于数据量标注画像可信度 |
| **对比基准** | 与同龄/同类型患者群体对比 |
| **预测曲线** | 基于当前轨迹预测未来能力走势 |
| **数据来源权重** | 不同来源数据赋予不同评估权重 |

### 6.2 画像更新策略

| 数据类型 | 更新频率 | 算法 |
|----------|---------|------|
| 实时指标（情绪、注意力） | 每次数据采集后 | 移动加权平均 |
| 日常表现（任务完成） | 每日 | 指数平滑 |
| 能力等级 | 每周 | 多维加权聚合 |
| 全面评估 | 每月 | 专家复核 + AI校准 |
