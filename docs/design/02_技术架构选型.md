## 2. 技术栈选型

### 2.1 MVP阶段（快速验证）

**原则：快速开发、易于验证、成本低**

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| **前端** | React (Web端) + React Native (移动端) | 跨平台，快速开发 |
| **后端** | Node.js + Express/Koa | 轻量级，快速原型 |
| **数据库** | PostgreSQL | 开源，关系型数据友好 |
| **AI服务** | 调用通用大模型API | 避免从头训练 |
| **部署** | 云服务器 (阿里云/腾讯云) | 快速部署 |

### 2.2 核心框架阶段（扩展优化）

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| **前端** | React / Vue (Web端) | 更成熟的生态 |
| **后端** | Go / Java (可选) | 性能优化 |
| **数据库** | PostgreSQL + Redis | 缓存提升 |
| **AI** | 自研轻量级模型 + 微调 | 降低API依赖 |
| **多智能体** | LangGraph / AutoGen | 智能体协作框架 |

### 2.3 生态系统阶段（规模化）

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| **微服务** | Kubernetes / Docker | 容器化部署 |
| **消息队列** | Kafka / RabbitMQ | 高并发处理 |
| **数据湖** | 大数据平台 | 多源数据处理 |
| **AI训练** | PyTorch / TensorFlow | 自研训练 |

## 3. 推荐MVP技术栈（立即可用）

### 前端技术
- **React 18** - 组件化开发
- **Tailwind CSS** - 快速UI构建
- **React Router** - 路由管理

### 后端技术
- **Node.js 20+** - 运行环境
- **Express** - Web框架
- **Prisma** - ORM数据库操作
- **Socket.io** - 实时通信（可选）

### 数据库
- **PostgreSQL 15+** - 主数据库
- **SQLite** (开发阶段) - 轻量级

### AI集成
- **OpenAI API** / **Claude API** - 初始推理
- **LangChain** - AI应用开发框架

## 5. 开发优先级

### Phase 0: 准备阶段
1. 环境搭建
2. 数据库设计
3. 基础工程配置

### Phase 1: MVP最小功能
1. 患者/老师登录
2. 简单的数据录入界面
3. 基础能力画像展示
4. 成长曲线可视化

### Phase 2: 咖啡馆场景
1. 任务管理界面
2. 康复老师记录工具
3. 实时数据反馈

### Phase 3: 画像引擎
1. 简单画像算法
2. 多源数据融合
